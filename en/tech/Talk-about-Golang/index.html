The generation of Go language

According to Rob Pike’s speech in San Francisco, in 2007 Rob Pike did some trivial but core work for Google’s huge C++ program. Due to the heavy weight of C++, it was required on Google’s huge distributed compilation cluster. Take 45 minutes.

Rob Pike believes that C++ has too many features, and simplifying the language must be a greater achievement, so Rob Pike and Robert Griesemer discussed and optimized C++. They found Uncle Ken and convinced him to work together To do something, they want to create a new development language and don’t want to use C++ anymore.

Rob Pike kept thinking on the way home, so he wrote an email to Uncle Ken and Robert Griesemer describing the naming and method of Go:

Robert: Using the C language as a prototype, repair some obvious defects, remove garbage functions, and add some missing functions.
Rob: The name “Go” has many advantages. The name is very short and easy to spell. Tools can be called: goc, gol, goa. If you have an interactive debugger/interpreter, you can simply call it “Go” and use the suffix .go.
**Robert: Define empty interface syntax: interface{}. *All interfaces are implemented through this, so it can be used instead of void.

In the speech, Rob Pike also mentioned that although the Go language is based on C as the prototype and designed with C++ as the target, not many C++ programmers have switched to Go language development. In fact, most Go programmers are from Python. Turning to Ruby, there are few C++ programmers among them. He thinks that the design of Go and C++ may be too different. C++ hopes that all solutions can be easily obtained. This way of thinking is different from the way Go works. . Zero cost is not the goal, at least zero CPU cost is not. **Go’s goal is to liberate programmers. **

Go is not all-encompassing, you don’t expect it to have everything, and you can’t precisely control every detail. For example: Go does not have RAII, but you can use a garbage cleaner instead, although you can’t even use the memory release function.

You can get a lot of easy-to-understand but powerful toolsets from Go to combine solutions to problems. It may not be as fast or refined as other languages, but it must be easier to write, easier to read, easier to understand, and perhaps safer.

From another perspective, Go must be more simplified:

Python and Ruby programmers turn to Go because they don’t need to learn more and care but can get better performance, and even use concurrency features.

**C++ programmers are reluctant to switch to Go because they are doing their best to fully control their programs and do not want to change this way. For them, software is not just about getting the job done, but about doing the job well. For those who use scripting languages, the focus is on realization and creation, which is why. **

Advantages of Go language

First of all, the Go language is a brand-new language designed to make full use of modern hardware performance while taking into account development efficiency. It is a cross-platform (Mac OS, Windows, Linux, etc.) statically compiled language. At the same time, taking into account the powerful performance of C++ and the development efficiency of the scripting language, according to a survey of Go language users, it is found that the most used Go language in the world is actually China, which may also be related to the growing Chinese market.

The Go language was born to do large-scale projects. The current Go language is mainly used in the development of back-end services. With the improvement of the Go project in the future, it will be used in *systems, games, UI interfaces, AI, Internet of Things and other fields *, will be widely used. More and more people use Go language instead of scripting language, and use Go language instead of heavy C++ for game servers. Due to the inherent high concurrency of the Go language, most of the research on blockchain uses the Go language, just as most research crawlers use Python.

For my 10,000-year Java dog, I recently encountered a bottleneck in WEB development. Although Spring Boot2+Spring Cloud+Docker solved the problem of rapid development and rapid deployment, due to the serious shrinkage of the company’s information cost (each non- Problems encountered by the information department of Internet companies), existing equipment is outdated, the system is not expandable, the information island is serious, etc., which leads me to find a maximum squeezing of hardware devices, and at the same time** Lightweight** language** for system refactoring**.

At this time, the Go language jumped out. Why can’t a system language be used as a web application? Because of the development of scripting languages ​​and microservices, more and more lightweight modules + restful APIs are used to separate the front and backends. So, isn’t the development of APIs that support rest a natural advantage of Go? As for the front-end and back-end data exchange, the template engine of the Go language is no worse than the mainstream freemarker.

Startup speed

This theory is not about comparing languages, but web containers. Because Go language is directly packaged into a binary executable file, its startup is a system-level startup. Compared with containers in the industry, it does not require data support at all, and it can be seen with the naked eye.

Multithreading and coroutines

Due to the performance overflow of modern hardware CPUs, the significance of concurrent programming is self-evident. Of course, many languages ​​support multi-threaded, multi-process programming, but unfortunately, it is not so easy and pleasant to implement and control. The difference between the Go language is that the language level supports goroutine concurrency (coroutines are also called microthreads, which are lighter, less expensive, and have higher performance than threads), and they are very simple to operate. The language level provides keywords (go ) Is used to start the coroutine, and thousands of coroutines can be started on the same machine.

Get rid of the evil try…catch

Anyone who develops in Java has to complain about the evil try…catch, which is disgusting, bloated, and has to write. Go language uses error as the basic data type, and no longer supports the use of try…catch at the language level. This should be regarded as a very bold and revolutionary initiative. It is no wonder that many people complain about the indecentness of the GO language. However, out of the traditional concept, the developers of GO believe that in the process of programming, it is very important to ensure the robustness and stability of the program, and the precise handling of exceptions is very important. Only after each logical processing is completed, the upper layer is clearly informed. Call, whether there is an exception, and the upper layer calls to deal with the exception in a clear and timely manner, so that the robustness and stability of the program can be guaranteed to a high degree. Function has multiple return values

In Java, when we write functions to perform return, if multiple values ​​are involved, they often need to be encapsulated in Map. At this point, Go draws on languages ​​such as Python and supports multiple return values. At the same time, because of the addition of anonymous functions, Make the whole call process wonderful. High-performance HTTPServer

As a server language appearing in the Internet age, the ability to serve users is indispensable. GO has its own HTTP/TCP/UDP high-performance server at the language level, based on coroutine concurrency, to provide the most direct and effective capability support for business development. To implement a high-performance HTTP Server in the GO language, it only takes a few lines of code to complete it. It is very simple. The startup speed kills other WEB containers. At the same time, the maintenance cost is extremely low and there is no external dependency.

Mandatory programming standards

The programming specifications of the GO language are mandatory to be integrated into the language, such as clearly specifying the placement of curly braces, mandating one sentence per line, not allowing unused packages to be imported, not allowing unused variables to be defined, providing gofmt tools to force formatting codes, etc. .

This point is very dissatisfied with some programmers. Some people have published N crimes of the GO language, and there is no lack of accusations against programming standards. But for me or from the perspective of engineering management, any development team will formulate specific programming specifications for a specific language, especially for companies like Google. The designers of GO believe that instead of writing the specification in the document, it is better to force the integration into the language, which is more direct and makes use of teamwork and project management.

When I’m old, I don’t like all kinds of elegant writing. Instead of complying with a public document, it is better to report an error directly. For example, in the Go language, if you import a certain package, or declare a certain A variable, but not used in the follow-up, Go will directly report an error when compiling, which is good.

Install and use Go language in Mac OS

Download and install

Installing the Go language in Mac OS is very simple, just go to Official website to download the pkg package. However, due to some well-known reasons, this website is often inaccessible. But the download address is not walled. Here I released the latest v1.12 version of download address

After the installation is completed, the environment variables are automatically configured, and there is no need to modify the configuration files (except GOBIN). Go development can already be carried out at this time, but if you need to use a third-party framework such as beego, build You also need to configure the path of GOBIN, use go version to view the current version, and use go env to view the current configuration information

Environment variable configuration

  1. Open the terminal, cd ~ to enter the home directory

  2. ls -all view all files to see if there is a .bash_profile file

  3. If it does not exist, execute touch .bash_profile to create a new

  4. If it exists, execute vi .bash_profile to open for editing

Add the following parameters in the file

  1. Save the edit, and then execute source ~/.bash_profile to complete the configuration of environment variables
  2. go env to view the environment variables after successful configuration

PS: If my Mac OS environment is configured and it still takes effect after restarting, I must re-execute source ~/.bash_profile. Later, I found out that zsh was installed to make the terminal look more attractive, so the system loaded by default is ~/.zshrc file, so after configuring the environment variables, you need to add a line of source ~/.bash_profile at the end of the ~/.zshrc file so that there is no problem.
